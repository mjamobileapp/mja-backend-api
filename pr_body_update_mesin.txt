Perubahan fungsi updateMesin untuk mendukung sinkronisasi dual relay (Washer & Dryer) menggunakan Database Transaction (Upsert/Delete).

## Perubahan pada Model (`src/models/mesin.js`)
- Menggunakan Database Transaction (`beginTransaction`, `commit`, `rollback`)
- Parameter fungsi berubah dari `(id, body)` menjadi `(espIdParam, body, updatedBy)`
- Menambahkan fungsi internal `syncMesin(jenis, isAktif, channelPin)`:
  - `isAktif === 1`: Cek jika mesin sudah ada → UPDATE (`cabangId`, `updatedBy`, `updatedDate`), jika belum → INSERT
  - `isAktif === 0`: DELETE record dari database
- Mengambil `namaMesin` asli dari database agar tidak berubah saat update

## Perubahan pada Controller (`src/controller/mesin.js`)
- Validasi fields required: `idMitra`, `cabangId`, `espId`
- Validasi washer & dryer harus bernilai 0 atau 1
- `updatedBy` diambil dari `req.user.username || req.user.id`
- Error handling untuk error baru: `"Modul mesin tidak ditemukan di sistem."` (404)

## Endpoint
**PUT** `/api/backoffice/mesin/:espId`

```json
{
  "idMitra": 9,
  "cabangId": 4,
  "espId": "A4:CF:12:3A:5B:7C",
  "washer": 1,
  "dryer": 0
}
```
